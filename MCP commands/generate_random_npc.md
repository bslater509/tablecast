# Generate Random NPC with Image via MCP Server

This document provides a universal prompt command that you can give to any AI assistant (such as Gemini, Claude, or ChatGPT) that has access to the Tablecast MCP server and possesses image-generation capabilities.

The prompt instructs the AI to generate a random D&D 5e NPC, create a custom portrait image for them, and save the complete record to the server using the `create_npc` tool—all without touching local files.

---

## The AI Prompt Command

Copy and paste the following prompt directly into your AI assistant:

```text
Please generate a completely random, detailed D&D 5e NPC with a portrait image, and register it to the Tablecast database using the MCP server tools.

Follow these steps precisely:
1. Generate the NPC details:
   - A unique fantasy name, race, class, level, alignment, and challenge rating (CR).
   - Core ability scores (Strength, Dexterity, Constitution, Intelligence, Wisdom, Charisma).
   - Narrative details: personality traits, physical appearance, backstory (history), and relationship with the party.
   - 1-3 combat actions (name, description, to-hit modifier, and damage formula).
   - Standard D&D 5e starter inventory items.

2. Generate a visual portrait:
   - Use your image generation tool (e.g., Imagen, DALL-E) to create a high-quality fantasy-style portrait of this NPC based on their appearance description.
   - Do NOT save this image locally. Instead, use the generated image's URL/URI.

3. Save the NPC to the server:
   - Use the `create_npc` tool from the MCP server.
   - Populate the fields including `name`, `race`, `class`, `level`, `hp`, `maxHp`, `ac`, `cr`, `strength`, `dexterity`, `constitution`, `intelligence`, `wisdom`, `charisma`, `inventory`, `actions`, `description`, `alignment`, `appearance`, `personality`, `history`, and `partyRelationship`.
   - Set the `imageUrl` and `largeImageUrl` parameters to the URL of the portrait image you generated.
```

## Tool Reference: `create_npc`

For your reference, here is the schema and parameters that the AI will use to call the `create_npc` tool:

| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `name` | string | **Yes** | NPC name. |
| `race` | string | No | NPC race. |
| `class` | string | No | NPC class. |
| `level` | number | No | NPC level (default: 1). |
| `hp` | number | No | Current hit points (default: 10). |
| `maxHp` | number | No | Maximum hit points (default: 10). |
| `ac` | number | No | Armor Class (default: 10). |
| `cr` | string | No | Challenge Rating, e.g., '1/4', '1', '5' (default: '0'). |
| `imageUrl` | string | No | NPC image/avatar URL. |
| `largeImageUrl` | string | No | NPC large portrait URL. |
| `strength` | number | No | Strength score (default: 10). |
| `dexterity` | number | No | Dexterity score (default: 10). |
| `constitution` | number | No | Constitution score (default: 10). |
| `intelligence` | number | No | Intelligence score (default: 10). |
| `wisdom` | number | No | Wisdom score (default: 10). |
| `charisma` | number | No | Charisma score (default: 10). |
| `inventory` | array | No | List of items: `[{ name: string, quantity: number, weight: number }]`. |
| `actions` | array | No | List of combat actions: `[{ name: string, description: string, toHit: number, damage: string }]`. |
| `description` | string | No | Narrative/GM description. |
| `alignment` | string | No | NPC alignment (e.g., Neutral Good). |
| `appearance` | string | No | Physical appearance details. |
| `personality` | string | No | Personality traits. |
| `history` | string | No | Backstory / Lore. |
| `partyRelationship` | string | No | Relationship with the adventuring party. |
| `isVisibleToPlayers` | boolean| No | Whether players can see this NPC in the wiki (default: false). |

## How it Works

1. **Universal Execution:** Because the instructions are written in natural language, any LLM (Gemini, Claude, GPT) that supports tool-use and is connected to the Tablecast MCP server can interpret and execute these instructions.
2. **Zero Local Footprint:** The image generated by the LLM is hosted on its generation service's temporary CDN/storage and passed directly as a URL to the `create_npc` tool. The database receives this URL, avoiding any local image download or file manipulation steps on the user's host machine.
3. **Automated Modifiers:** The Tablecast MCP server will automatically compute ability modifiers (e.g., Strength modifier from Strength score) upon insertion, ensuring the NPC sheet is fully functional in the UI immediately.
